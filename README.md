# SNH48 Live Enhancer

> 优化 SNH48 GROUP 公演直播网站（live.48.cn）体验的 Chrome 浏览器扩展。
>
> **当前版本**：3.0.0 · **Manifest**：MV3 · **适用站点**：<https://live.48.cn/*>

---

## 核心特性

| 分类 | 功能 | 说明 |
|------|------|------|
| 🎨 视觉 | **全站暗黑模式** | 公演贡献榜、成员卡片、列表页、播放器、输入框等均适配暗色主题 |
| 🎬 视频 | **播放增强** | 自定义快捷键（全屏 / 画中画 / 截图）、进度自动记忆、跨域检测提示 |
| 🔍 搜索 | **成员反向索引** | 搜索成员名即可检索其参与的公演；访问公演页时自动建立本地索引 |
| 📅 搜索 | **嵌入搜索入口** | 页面顶部一键搜索，支持分团筛选、键盘导航、结果 LRU 缓存 |
| 📅 日历 | **公演日历视图** | 基于索引数据按月查看，SNH48/GNZ48/BEJ48/CKG48/CGT48 分团筛选 |
| 🔔 提醒 | **公演开播提醒** | 即将开始的公演桌面通知，支持自定义提前分钟数 |
| 🧭 导航 | **分团快捷导航** | 页面底部常驻导航栏，一键切换各团主页 |
| 🎛 UI | **头部隐藏** | 观看页面可一键折叠顶部导航，腾出更多播放区域 |
| 📜 历史 | **观看历史记录** | 自动记录访问的公演页，搜索结果中标注已观看 |
| ⚙️ 配置 | **导入/导出** | 一键备份 / 恢复所有扩展配置与自定义快捷键 |

## 快速开始

```bash
# 1. 克隆或下载本项目到本地
# 2. 打开 Chrome 扩展页面：
chrome://extensions/

# 3. 右上角开启「开发者模式」
# 4. 点击「加载已解压的扩展程序」，选择项目根目录：
#    snh48-live-enhancer/  ← 必须包含 manifest.json
# 5. 访问 https://live.48.cn/ 即可使用
```

> **首次使用**：页面加载后，右下角会出现浮动面板，可在其中开启功能并手动构建索引。

## 权限说明

| 权限 | 用途 |
|------|------|
| `storage` / `unlimitedStorage` | 保存用户配置、成员-公演索引、观看历史、播放进度 |
| `notifications` | 公演开始前推送桌面通知 |
| `alarms` | 精确时间触发提醒 |
| `activeTab` | 注入快捷操作所需当前标签页上下文 |
| `tabs` | 配置变更后同步到已打开的 live.48.cn 标签 |
| `contextMenus` | 右键菜单索引功能 |
| `https://live.48.cn/*` | **插件唯一作用域**，不会访问其他站点 |

> 本扩展**不收集、不上传**任何用户数据，索引与配置均存储于本地浏览器。

## 使用指南

### 快捷键

| 操作 | 默认快捷键 | 可否自定义 |
|------|----------|-----------|
| 暂停 / 播放 | `Space` | 是 |
| 全屏切换 | `F` | 是 |
| 画中画 | `P` | 是 |
| 截图 | `S` | 是 |
| 打开嵌入搜索 | `Ctrl + K` | — |

> 在浮动面板「快捷键」区域可随时修改并保存自定义绑定。

### 构建索引

扩展提供两种索引模式，可在浮动面板操作：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **访问自动索引** | 每次进入公演详情页自动解析并存入索引 | 日常使用，数据渐进式增长 |
| **批量索引** | 指定公演 ID 范围（如 `1 ~ 10000`）一次性扫描建立完整索引 | 首次使用需覆盖历史公演 |

批量索引的内部机制：
- 3 并发抓取，自动识别每个 ID 归属的团（club/1=SNH48 / club/3=GNZ48 …）
- 每秒最多 5 次请求，遇 HTTP 429 自动退避 5 秒后重试
- `indexedAt` 时间戳驱动去重，相同 URL 不会重复插入
- 完成后索引会被写入 `chrome.storage.local` 持久化

### 查看日历

浮动面板点击「日历」：
- 默认显示当月，可左右翻月或点击「今天」回到当月
- 顶部有 SNH48 / GNZ48 / BEJ48 / CKG48 / CGT48 分团筛选按钮
- 有公演的日期会以高亮色标出，点击可展开当日公演列表
- 点击某场公演可跳转到对应页面

## 常见问题

**Q：为什么打开 live.48.cn 后没有出现插件 UI？**
A：首次加载需等待 `document_idle` 就绪。请刷新一次页面，或检查 `chrome://extensions/` 中本扩展是否被启用 / 被禁用。

**Q：成员搜索找不到某场公演？**
A：索引是在访问公演页时建立的。浮动面板可按 ID 范围批量扫描。v3.0.0 起：
- 容量已提升至 50000 场（含分片归档）
- 旧数据不会被裁剪丢弃
- 新增增量索引模式：对已索引 ID 自动跳过

**Q：空格键无法控制视频暂停 / 播放？**
A：事件监听位于**捕获阶段**，优先于网站播放器处理。若无效，请：
1. 确认「键盘快捷键」开关已开启
2. 确认当前焦点不在输入框内
3. 视频元素位于 Shadow DOM 中时已自动处理

**Q：暗黑模式下某些元素仍是亮色？**
A：欢迎在仓库提交 Issue，并附上**元素截图**和**DOM 类名**，我们会持续补充覆盖。

**Q：截图失败并提示「跨域限制」？**
A：部分视频源由第三方 CDN 提供，浏览器安全策略禁止 canvas 截图。v3.0.0 起会给出明确提示，不影响其他功能。

**Q：如何备份 / 迁移扩展配置？**
A：浮动面板 → 「导出配置」生成 JSON 文件。新设备上同位置选择「导入配置」即可恢复设置（含自定义快捷键）。

**Q：刷新页面后视频从头播放？**
A：v3.0.0 起自动记忆播放进度（每 10 秒保存一次）。刷新后会自动跳转到上次位置并 Toast 提示。播放完毕后进度自动清除。

**Q：Service Worker 被杀后进度会丢吗？**
A：`indexingState` 已持久化到 `chrome.storage.session`，SW 重启后自动检测并恢复未完成的批量索引任务。

## 设计与实现要点

### CSS 变量系统

在 `enhancer.css` 顶部定义了浅色 / 深色两套 CSS 变量，由 `html.snh48-dark` 选择器切换。相比硬编码颜色值，CSS 变量便于主题扩展与调试。

### 消息协议

所有 content ↔ background 消息类型统一在 `message-types.js` 中定义为常量，附带 JSDoc 说明请求 / 响应结构。background 侧由 `message-handler.js` 统一路由，避免 `if-else` 散落。

### URL 压缩

`index-store.js` 中对公演 URL 做前缀压缩（如 `https://live.48.cn/Index/invideo/club/6/id/7553` → `/c/6/7553`），节省 50%~70% 存储空间，`getIndexStats()` 可查看压缩率。

### Club ID 映射

经对 live.48.cn 站点实际 HTML 校验，各团的 club ID 为：

| Club ID | 团体 |
|---------|------|
| 1 | SNH48 |
| 2 | BEJ48 |
| 3 | GNZ48 |
| 4 | SHY48（已解散，URL 仍可访问） |
| 5 | CKG48 |
| 6 | CGT48 |

`calendar-view.js`、`index-crawler.js`、`quick-nav.js` 均使用同一套映射，保持一致。

### v3.0.0 · 2026-06-11

**架构重构**
- `content.js`：~2551 行拆分为 9 个功能模块 + 108 行入口
- `background.js`：~1429 行拆分为 4 个功能模块 + 20 行入口

- 搜索项 / 历史项：`innerHTML` 拼接 → DOM API 构建，消除 XSS 风险

**新功能**
- 播放进度记忆（10 秒间隔保存 / LRU 最多 100 条）
- 观看历史记录与搜索结果标注

- 批量索引增量模式

**性能优化**
- MutationObserver 50ms 防抖 + WeakSet 缓存
- 归档分片 `storage.get` 批量加载
- 搜索结果 DOM diff（URL 集合对比）

- 索引 URL 压缩（前缀替换）

**Bug 修复**
